Agilo for Scrum is an open source, web-based tool to support the Scrum process. It is developed and maintained by agile42, and supports Scrum Teams, Scrum Master, Product Owner and the main ceremonies and artifacts. It streamlines agile and lean software development by increasing transparency, velocity and visibility while reducing project overhead.

Scrum Team
Scrum Master
Product Owner

Artifacts

Burn Down Chart

Sprint Backlog

The concept of Agilo for Scrum is based on traceability: All items like requirement, user story, task etc. are linkable and therefor bi-directional traceable.
Agilo is based on Trac a widespread Ticket Tracking System, and can be freely downloaded, used as SaaS or as a Virtual appliance.
Agilo for Scrum is written in the Python programming language. Since January 2008 it is available under the Apache Software License 2.0.

[edit] Features

Real Ticket Typing: Agilo for Scrum allows to define which attributes belongs to which defined type of ticket and consistent customization.
Ticket References and Linking: Agilo supports the defined Scrum process including the semantic behind the linking of two different items. (a user story ticket can be linked with a set of task incl. traceability related to e.g. changes)
Sprint Data and Burndown chart: Maintaining the Product Backlog updated and sorted according to Business Value; a quick glimpse on the selected Product Backlog for a specific Sprint; accessing the Burn down chart for a specific Sprint
Multi Project Functionality
